Conditions for liability shift
Card scheme rules
↻
Condition
Visa, Mastercard and other schemes each define their own rules for when liability shifts. Not all schemes treat every transaction type the same way.
Authentication method
↻
Condition
The type of authentication used matters. Full 3DS authentication typically triggers a shift; frictionless flows or exemptions may not in all cases.
Proper transaction configuration
↻
Condition
The merchant must correctly implement and pass 3DS data in the transaction. Misconfigured or incomplete data can void the liability shift.
3DS authentication can shift fraud liability to the issuing bank — but only when all conditions are correctly met.
